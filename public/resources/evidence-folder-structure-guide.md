# Evidence Folder Structure Guide

ComplianceOne stores evidence references. Keep the source evidence in your approved business repository and record the location in the relevant control.

## Suggested folder structure

```text
CPCSC Readiness/
  00-Readme-and-Scope/
  01-Access-Control/
  02-Awareness-and-Training/
  03-Audit-and-Accountability/
  04-Configuration-Management/
  05-Identification-and-Authentication/
  06-Incident-Response/
  07-Maintenance/
  08-Media-Protection/
  09-Physical-Protection/
  10-Personnel-Security/
  11-Risk-Assessment/
  12-Security-Assessment/
  13-System-and-Communications-Protection/
  14-System-and-Information-Integrity/
  90-Reports-and-Exports/
```

## Evidence naming convention

```text
YYYY-MM-DD_Control-Number_Evidence-Type_Description_Owner
```

Example:

```text
2026-04-29_Control-01_Access-Review_Q2-User-Access-Review_IT-Ops.xlsx
```

## Evidence record checklist

- Evidence title:
- Control supported:
- Evidence type:
- Storage path/reference:
- Owner:
- Date created:
- Date reviewed:
- What this evidence proves:
